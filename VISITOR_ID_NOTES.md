# visitor_id 기반 채팅 분리

이번 수정본은 동일한 `admin` 계정으로 로그인하더라도 브라우저마다 별도의 `visitor_id` 쿠키를 발급해
각자 자기 채팅만 보이도록 변경한 버전입니다.

## 핵심 동작
- 로그인은 계속 공용 `admin` 계정으로 수행
- 채팅 목록 / 생성 / 조회 / 삭제 / 메시지 전송은 `admin + visitor_id` 기준으로 분리
- `visitor_id`는 HttpOnly 쿠키로 관리

## 주의
- 기존 공용 채팅 기록은 새 브라우저별 채팅과 분리되어 보이지 않을 수 있습니다.
- 같은 PC의 같은 브라우저를 여러 사람이 같이 사용하면 채팅도 같이 보입니다.
