# Phase24 - ChatGPT 스타일 진행 상태 UI 개선

## 목적

기존 Phase23에서는 RAG 내부 프로세스가 모두 큰 타임라인 형태로 기본 노출되었습니다.
이 방식은 디버깅에는 좋지만, 실제 사용자 화면에서는 답변보다 프로세스 로그가 더 강조되어 ChatGPT 같은 자연스러운 사용감과는 차이가 있었습니다.

이번 Phase24에서는 진행 상태를 다음 방식으로 변경했습니다.

1. 기본 화면에서는 현재 상태를 한 줄 문구로만 표시합니다.
2. 답변이 스트리밍되기 시작하면 답변 본문을 우선 보여줍니다.
3. 상세 검색 과정은 `검색 과정 보기` 버튼을 눌렀을 때만 펼쳐집니다.
4. 답변 완료 후에도 해당 답변에 대한 검색 과정은 현재 화면에서 확인할 수 있습니다.

## 변경 파일

- `my-app/src/components/chat-screen.tsx`
- `my-app/src/app/globals.css`

## 사용자에게 보이는 상태 문구 예시

- 관련 이력을 검색하고 있습니다...
- 가장 관련 높은 이력을 선별하고 있습니다...
- 답변을 정리하고 있습니다...
- 답변을 작성하고 있습니다...

## 유지되는 기능

- 백엔드의 process 이벤트 스트리밍은 유지됩니다.
- 상세 RAG 단계는 `검색 과정 보기`에서 확인할 수 있습니다.
- 텍스트 답변 스트리밍은 그대로 유지됩니다.
- 참조 표/기본 정보는 기존처럼 마지막에 한 번에 붙습니다.

## 참고

이번 변경은 주로 프론트 UI 표현 방식 변경입니다.
백엔드 RAG 로직, 검색 로직, 스트리밍 이벤트 구조는 변경하지 않았습니다.
